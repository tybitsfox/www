DROP DATABASE

Name

DROP DATABASE -- 删除一个数据库

Synopsis

DROP DATABASE name

描述

DROP DATABASE 删除一个现存数据库的目录入口并且删除包含数据的目录。 只有数据库所有者能够执行这条命令。 还有，如果你或者任何其它人正在与目标数据库连接，那么就不能执行这条命令。 （与 postgres 或者任何其它数据库连接，再发出这条命令。）

DROP DATABASE 不能撤销，小心使用！

参数

name

要被删除的现有数据库名。

注意

这条命令在和目标数据库联接时不能执行。 通常更好的做法是用 dropdb, 程序代替，该程序是此命令的一个封装。

兼容性

在 SQL 标准中没有 DROP DATABASE 语句。

又见

CREATE DATABASE
