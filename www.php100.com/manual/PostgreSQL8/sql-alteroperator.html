ALTER OPERATOR

Name

ALTER OPERATOR -- 改变一个操作符的定义

Synopsis

ALTER OPERATOR name ( { lefttype | NONE } , { righttype | NONE } ) OWNER TO newowner

描述

ALTER OPERATOR 改变一个操作符的定义。目前唯一能用的功能是改变操作符的所有者。

要使用 ALTER OPERATOR，你必须拥有操作符表。 要修改所有者，你必须是新的所有角色的直接或者间接成员， 并且该角色在操作符表的模式上必须有 CREATE 权限。 （这些限制就保证了修改所有者所能干的事情不比删除和重建操作符表更多。 不过，超级用户可以以任何方式修改任意操作符表的所有权。）

参数

name

一个现有操作符的名字（可以有模式修饰）。

lefttype

操作符的左操作数的数据类型；如果它没有左操作数，那么写 NONE。

righttype

操作符的右操作数的数据类型；如果它没有右操作数，那么写 NONE。

newowner

操作符的新所有者。

例子

改变一个用于 text 的用户定义操作符 a @@ b 的所有者： 

ALTER OPERATOR @@ (text, text) OWNER TO joe;

兼容性

在 SQL 标准里没有 ALTER OPERATOR 语句。

又见

CREATE OPERATOR, DROP OPERATOR
