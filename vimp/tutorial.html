快速入门教程

 tutorial

如果你刚刚开始使用 Vimperator （也就是说，没有作任何定制），你应该是在一个荒凉的窗口查看这份帮助页面。菜单栏、导航栏和书签栏都被隐藏了。如果你没有看到 介绍 中的警告，你可以使用这条命令恢复它们：

 :set go+=mTB<CR>

其中 <CR> 表示按下 <Enter> 或者 <Return> 键（回车）。如果你是 Vim 的老用户，这些看起来可能很熟悉，这正是我们所希望的。

但是依作者的观点，熟悉 Vimperator 最好的方式就是暂时让它们保持隐藏的状态（重新隐藏它们： :set go=<CR> ）。你可以在 选项帮助 的 'guioption' 条目中获取关于图形界面设置的更多信息。

Vimperator 的模式

 modal

Vimperator 的力量，和 Vim 一样，来自于它的模式。键盘输入的含义会因浏览器所处的模式不同而发生变化。 Vimperator 有多个模式，其中最重要的两个是“普通”模式和“命令行”模式。

Vimperator 启动时，默认会进入普通模式。绝大部分时间你都会处于这个模式中。

另一个 Vimperator 的核心模式，命令行模式，可以通过在普通模式下输入 ':' （冒号）进入。你会经常看到 Vimperator 的命令由 ':' 开始，这表明紧接着是一条命令。

按下 <ESC> 会从命令行模式返回普通模式。按下 <ESC> 会从 Vimperator 的大多数其他模式中返回普通模式。

获得帮助

 getting-help

Vim 是一个很棒的编辑器但它不是一个浏览器。因此，即使是用过 Vim 多年的老用户也许迟早也会需要查看 Vimperator 的帮助文档。大多数 Vimperator 特性的文档都可以方便的通过 :help 命令访问。例如，你可以用：

 :help :help<CR>

找到关于 :help 命令的帮助。

类似的，可配置选项的帮助可以通过 :help '{选项名}' 得到（和 Vim 一样，选项名需要用单引号括起来）。要得到所有可用选项的信息当然是使用 :help options[c] 。

你可以用：

 :help gt<CR>
 :help gT<CR>

得到关于映射 gt 和 gT 的信息。

最后，除了帮助系统，:exusage、:viusage 和 :optionusage 命令都可以作为快速参考。

无鼠标流

 living-mouseless

— 或者说，我是如何学会了不再担忧并且彻底爱上了这块键盘的。

和传奇的 Vim 一样，Vimperator 的效率来自于用户的双手无需离开键盘就能完成绝大多数工作。在某些领域，比如图形界面设计或者游戏，鼠标有着明显的优势。但 Vimperator 证明了使用浏览器上网冲浪并不是其中之一。

以下是一些被普遍认为鼠标有优势的领域，看看 Vimperator 是如何挑战这些成见的。

滚动窗口

 keyboard-scrolling

Vimperator 中，用键盘滚动浏览器窗口是一件很简单的事：

  • j/k — 将浏览器窗口向下或向上滚动一行

  • h/l — 将浏览器窗口向左或向右滚动

  • <Space>/<C-b> — 向下或向上翻一页

  • <C-d>/<C-u> — 向下或向上翻半页

标准方向键（<Up>/<Down>/<PgUp>/<PgDn>）仍然可以正常工作。

历史记录与标签页

 tab-navigation history-navigation

浏览（例如，“前进”和“后退”）的完成和滚动窗口类似：

  • <C-o>/<C-i> — 在当前窗口或标签页的历史记录中前进或后退

习惯在 Vim 中使用标签的用户会发现切换标签页的键盘操作非常面熟：

  • gt/<C-n> — 切换到下一个标签页

  • gT/<C-p> — 切换到上一个标签页

  • g0/g$ — 切换到第一个或是最后一个标签页

  • d — 关闭当前标签页（删除缓冲区）

使用 :tabopen {url} 可以在新标签页中打开页面。使用 :open 可以在当前标签页中打开链接。普通模式下，t 键和 o 键分别被映射到这两个命令上，因此下面两对命令是等价的：

 :open my.webmail.com<CR>
 omy.webmail.com<CR>

 :tabopen vimperator.org<CR>
 tvimperator.org<CR>

冲浪提示

 hints-tutorial

你现在应该会用 Vimperator 打开网页了。但是等一下……如何 打开 页面上的链接呢？没有了那个长尾巴的家伙我应该如何“单击”它们呢？

答案就是 "hints" （提示）。进入提示模式后， Vimperator 会在每个找到的链接旁加上一个数字。想要打开一个链接，只需要输入相应的数字即可。

对于文字链接，还有一个更简单的办法：你可以直接输入链接标题中的文本， Vimperator 会找出所有包含该文本的链接并标识这些链接，这样就能缩小选择范围。如果你输入的文本唯一确定了一个链接， Vimperator 会直接打开这个链接。

无论用什么方法，只要 Vimperator 高亮了你想打开的链接，按下回车 <Enter> 你就可以打开它。

最常用的提示模式叫做 快速提示模式。你可以按 f 或 F 进入快速提示模式。小写的 f 会在当前标签页中打开目标链接，大写的 F 会在新标签页中打开目标链接。

你可以用 Vimperator 主页 来测试一下。使用 f 或 F 进入快速提示模式，高亮所有可见的链接，然后输入链接的文字。输入的文字应该很快就可以唯一确定你想要的链接，然后 Vimperator 会直接打开它。测试完之后记得用 <C-o> （后退）或者 d （删除缓冲区）回到这里，当然，用哪种方式取决与你是如何进入快速提示模式的。

常见问题

 common-issues

比如，你正在输入一个新的 URL，但半路想起来这个页面已经在上一个标签页中打开了。你的命令行可能看起来像这样：

 :open my.partial.url/fooba

这事你可以退出命令行然后用以下命令打开已经加载了的页面：

 <Esc>gT

保存设置 —— vimperatorrc

 vimperatorrc

当你为 Vimperator 定制了自己的选项、映射和命令之后，你会希望下次打开 Vimperator 时它们依然可用。和 Vim 相似，我们有 vimperatorrc 文件。

使用 :mkv 命令保存当前的设置，它们会在下次 Vimperator 启动时被自动加载。

该命令会生成 $HOME/.vimperatorrc 文件来保存你的设置。这是一个纯文本文件，和 vimrc 一样，你可以手工编辑它来更改配置。

快速退出

 quitting-without-menus

Vimperator 支持所有经典的 Vim 退出方式。

  • :xall — 缺省，命令行方式退出，保存浏览器会话

  • :qall — 命令行方式退出但 不会 保存浏览器会话

  • ZZ — 普通模式下直接退出，等价于 :xall

  • ZQ — 普通模式下直接退出，等价于 :qall

Firefox 上哪儿去了？

 whither-firefox

你可能现在有点找不着北了，没关系，你还是在使用 Firefox 。 Vimperator 很多时候还是会调用 Firefox 的功能。请参考 :help 得到如何使用这些命令和映射的更多信息。

  • :dialog — 你可以使用 :dialog 命令打开 Firefox 众多对话框中的一部分。请参考 :help :dialog 。

  • :bmarks — Vimperator 提供了新的方式来访问书签，但无论如何它们还是标准的 Firefox 书签。:bmark 会添加一个新的书签，:bmarks 会列出当前的所有书签。

  • :history — 该命令名副其实，会显示一个彩色的、可滚动、可点击的窗口，包含 Vimperator 的历史记录。

  • :emenu — 在 Vimperator 中使用命令行访问 Firefox 的菜单。

现在你可以自己摸索了。如果你使用了 :tabopen 命令，记得用 gt 或 gT 回到这个页面。如果你使用了 :open 命令，记得使用历史记录相关的快捷键（例如 H）来返回。如果你完全迷失了，还可以输入 :help<CR> 并单击“入门教程”链接回到这里。
