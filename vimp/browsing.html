surfing browsing

为了给 Vim 用户带来更好的浏览体验，Vimperator 拦截并重定义了几乎所有键盘输入。在少数情况下你可能希望能够将键盘输入传递给 Firefox 或是网页，使之保持原有的功能，有两种方法可以做到这一点：

pass-through <C-z> CTRL-Z

CTRL-Z

禁用除 <ES> 外的所有 Vimperator 快捷键，并将键盘输入传递给下一个 event handler （事件处理函数）。这在 Javascript 类表单（例如 Gmail 的 RichEdit 文本框）不工作时很有用。按下 <ESC> 退出此模式。如果在此模式下你需要把 <ESC> 传递给网页，先按 <C-v> 再按 <ESC>。

send-key <C-v> CTRL-V

CTRL-V

如果你只需要把一次击键传递给 Javascript 表单或者其他扩展，先按 <C-v> 再进行输入。这也可以用来使用一些被 Vimperator 屏蔽掉的 Firefox 快捷键，例如 <C-o>。当 Vimperator 被 <C-z> 暂时屏蔽时，<C-v> 会把下一个键盘输入传递给 Vimperator 而不是网页。

打开网页

 opening

o :o :open

:o[pen][!] [arg1], [arg2], …

o

在当前标签页中打开一个或多个 URL 。多个 URL 使用分隔符 'urlseparator' 分开（缺省为 ", "。注意：逗号后的空格是必须的）。第一个 URL 会在当前标签页中打开，其他 URL 会在新的标签页中打开。参数的解析顺序是：

  1. 若 URL 是一个存在的本地文件的文件名则打开本地文件，接受相对或绝对路径。

    • :open /etc/fstab 显示文件系统分区表

    • :open ../other/foo.html 打开文件（相对于家目录） /home/other/foo.html

  2. 如果参数看起来像是搜索用的关键字且第一个参数是搜索引擎名，则用指定的搜索引擎搜索之（:open wikipedia linus torvalds 会打开 Wikipedia 中 linus torvalds 的条目）。Vimperator 会根据搜索引擎的名字识别它的简写。你可以用 :dialog searchengines 设置搜索引擎的简写。

  3. 如果第一个参数不是搜索引擎名或其简写，则用缺省的搜索引擎搜索（由 'defsearch' 选项指定）。缺省情况下 :open linus torvalds 会打开 Google 并搜索 linus torvalds。

  4. 其他情况下，用 Firefox 打开（:open www.osnews.com, www.slashdot.org 会在当前标签页中打开 OSNews，并在新的背景标签页中打开 Slashdot）。

你可以用 :open -tags linux torvalds<Tab> 来补全标签是 "linux" 且包含关键字 "torvalds" 的书签。注意，只有标签页补全支持 "-tags" 参数，其他命令并不支持。能够使用 <Tab> 补全的对象可以通过 'complete' 选项指定。没有参数时，该命令会重新载入当前页面。使用 [!] 且没有参数时，重新载入当前页面并忽略缓存。

t :t :tabopen :tabnew :tabe :tabedit

:tabopen[!] [arg1], [arg2], …

t

和 :open 相似，只是会在新标签页中打开第一个 URL。使用 [!] 时， 'activate' 选项的 tabopen 值会被反转。

T

T

:tabd :tabduplicate

:[count]tabd[uplicate][!]

将当前标签页复制 [count] 次。最后一个克隆页是否被激活由 'activate' 选项的 tabopen 值决定。使用 [!] 时，tabopen 值会被反转。

O

O

参照当前页面地址在当前标签页中打开一个或多个 URL。和 o 相似，但会在 :open 命令行中预填当前页面的 URL。

:winopen :wopen :winedit

:wino[pen][!] [arg1], [arg2], …

和 :tabopen 相似，只是会在新窗口中打开页面。

<MiddleMouse> p

p

在当前标签页中打开剪贴板中的内容。使用 p，你可以选择一串非 URL 的文字（对于非 X11 用户：复制），然后使用（由 'defsearch' 选项指定的）缺省搜索引擎直接进行搜索。

P

P

在新标签页中打开剪贴板中的内容。和 p 类似，只是会在新标签页中打开页面。新缓冲区是否会被激活取决于 'activate' 选项。

gP

gP

在新标签页中打开剪贴板中的内容。和 P 类似，但会反转 'activate' 选项。

导航

 navigating

H <C-o> CTRL-O :ba :back

:[count]ba[ck] [url]

:ba[ck]!

CTRL-o

后退 [count] 次。如果指定了地址 [url]，则后退到第一个匹配的 URL。:back! 会跳转到浏览历史的最后。

L <C-i> CTRL-i :fo :fw :forward

:[count]fo[rward] [url]

:fo[rward]!

CTRL-i

前进 [count] 次。如果指定了地址 [url]，则前进到第一个匹配的 URL。:forward! 会跳转到浏览历史的最前。

:ju :jumps

:ju[mps]

列出所有可能的跳转，即当前标签页的浏览历史，也就是会话的历史。

当前浏览的页面会被用 ">" 标出，条目前的数字是配合 :back（在 0 以上的条目）或 :forward（在 0 以下的条目）命令使用的计数器。

gh

gh

gH

gH

在新标签页中打开我的主页。新标签页是否被激活取决与 'activate' 选项。

gu

[count]gu

访问第 [count] 层父目录。

对于 http://www.example.com/dir1/dir2/file.htm ，2gu 会打开 http://www.example.com/dir1/ 。

gU

gU

访问站点的根域名。

对于 http://www.example.com/dir1/dir2/file.htm ，2gu 会打开 http://www.example.com/ 。当浏览本地目录时，该命令会直接访问根目录。

写入

:w :write :sav :saveas

:sav[eas][!] [file]

退出

 save-session quitting

:q :quit

:q[uit]

关闭当前标签页。如果这是窗口中的最后一个标签页，窗口会被关闭。如果这是最后一个窗口，关闭 Vimperator。当 Vimperator 退出时，会话不会被保存。

:qa :qall :quita :quitall

:quita[ll]

退出 Vimperator。无论打开了多少标签页和或窗口都直接退出，不保存会话。

:wc :wclose :winc :winclose

:winc[lose]

关闭窗口。

:xa :xall :wq :wqa :wqall

:wqa[ll]

:xa[ll]

保存会话并退出。无论打开了多少标签或窗口都直接退出，保存会话。 :wq 命令和 Vim 中相应的指令的意义有所不同，它会关闭窗口而不是许多人要求的一个标签页。如果你觉得这样不爽，请在邮件列表中发信让我们知道。

ZQ

ZZ

ZZ
