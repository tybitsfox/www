options

Vimperator 有很多内置的变量和开关，它们的取值会产生许多特殊的效果。这些选项分为五类：

boolean 布尔型，可以是开（on）或是关（off）
number 数值型，值是一个数
string 字符串型，值是一个字符串
charlist 字符型，和字符串型类似，但仅由互不相同的字符组成
stringlist 字符串列表型，值是一列由逗号分隔的字符串

设置选项

 set-option.E764

:set :se

:se[t]

显示所有非缺省值的选项。

:se[t] all

显示所有选项。

E518 E519

:se[t] {option}?

显示选项 {option} 的值。

:se[t] {option} […]

对于布尔型选项，会将其打开（on）。

对于数值型选项，会显示选项的值。

对于字符串型选项，会显示选项的值。

:se[t] no{option} […]

对于布尔型选项，会将其关闭（off）。

:se[t] {option}! […]

:se[t] inv{option} […]

对于布尔型选项，会将选项的值反转。

:se[t] inv{option}={value} […]

:se[t] {option}!={value} […]

对于列表型选项，会设置或删除指定的值。

例如，已有：
 :set opt=foo,bar
然后设置：
 :set opt!=foo,baz
最后的结果将为：
 opt=bar,baz

:set-default

:se[t] {option}& […]

将选项重置为缺省值。

:se[t] all&

将所有选项重置为缺省值。

:set-args E487 E521

:se[t] {option}={value} […]

对于数值型选项，指定值必须为十进制。选项的当前值可以用 <Tab> 插入。

:set+=

:se[t] {option}+={value} […]

将数值型选项的值加上 {value} ，或是将 {value} 追加在字符串型选项的末尾。如果选项是字符串列表型，除非给定值为空，逗号会被自动添加。如果选项是一列标志（flag），多余标志会被自动删除。追加一个已经存在的标志，选项的值不会改变。

:set^=

:se[t] {option}^={value} […]

将数值型选项的值乘以 {value} ，或是将 {value} 加在字符串型选项的开头。如果选项是字符串列表型，除非给定值为空，逗号会被自动添加。

设置 Firefox 选项

 preferences firefox-options

Firefox 选项可以用以下命令查看和设置：

:prefs :preferences

:pref[erences]

显示 Firefox 的选项对话框。你可以在这个对话框中更改浏览器的设置。请注意，不是所有的选项都可以正常工作，因为 Vimperator 覆盖了一些键盘绑定并改变了 Firefox 的图形界面。

:prefs! :preferences!

:pref[erences]!

在当前标签页中打开 about:config 页面。在其中你可以修改 Firefox 的高级设置。

:set! :set-!

:se[t]! {preference}={value}

更改 Firefox 的 about:config 页面中的设置 {preference} 。你也可以用 :set! {preference}& 重置或者删除这些设置。

overridden-preferences

Vimperator 会在 Firefox 启动时设置几个选项。如果你不喜欢，可以用 :set! {preference}={value} 在你的 RC 文件中将它们设为其他的值。

设置的选项包括：

  • browser.startup.page

  • dom.popup_allowed_events

  • accessibility.typeaheadfind.autostart

  • accessibility.typeaheadfind

'ds' 'defsearch'

'defsearch' 'ds'
字符串型（缺省值： "google" ）

设置默认的搜索引擎。如果 :[tab]open [arg] 的参数 [arg] 既不像一个 URL 也不像一个搜索引擎或其别名，命令会调用缺省的搜索引擎搜索之。

这意味着如果将 'defsearch' 设置为 "youtube"， :open arnold schwarzenegger 和 :open youtube arnold schwarzenegger 效果将会完全相同。因此，你要么指定一个搜索引擎关键字来搜索要么直接调用 "youtube" 进行搜索。

如果 'defsearch' 选项为空，Firefox 会尝试打开参数 [arg]。

'hintmatching' 'hm'

改变提示模式下的提示匹配算法。可用的选项：

custom 使用 liberator.plugins.customHintMatcher(hintString) 指向的自定义函数。

'wildignore' 'wig'

'wildignore' 'wig'
字符串列表型（缺省值：""）

补全文件名时忽略的文件名模式列表。例如，:set wildignore=".\.o,\..\.s[a-z]{2}" 会忽略对象文件和 Vim 交换文件。

注意：Vim 中的匹配模式使用的是 glob 表达式而这里使用的是正则表达式。

'wim' 'wildmode'

'wildmode' 'wim'
字符串列表型（缺省值："list:full"）

命令行自动补全的工作模式。该选项的值是一个由逗号分隔成多个部分的字符串。每个部分分别指明了多次按下补全键（<Tab>）时的行为。第一部分对映第一次按下补全键时的动作，第二部分对映第二次的动作，以此类推。

每个部分中可用的选项：

"" 仅补全为第一个匹配项。
"full" 补全为下一个匹配项。最后一个匹配项后将恢复为原字符串。
"longest" 补全为最长的公共字符串。
"list" 当有多个匹配项时，将其全部列出。
"list:full" 当有多个匹配项时，将其全部列出并补全为第一个匹配项。
"list:longest" 当有多个匹配项时，将其全部列出并补全为最长的公共字符串。当仅由一个匹配项时，将其补全为唯一的那个匹配项。
