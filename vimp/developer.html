撰写文档

 documentation writing-docs

对于每个新特性，想要将补丁入库，撰写文档都是 必须的 。Vimperator 的文档使用 asciidoc 8.2.x 生成，位置在 src/locale/en-US/ 目录，可以用 make doc 编译。细节请参考 asciidoc documentation 。一般情况下你可以不考虑那些格式，文字也会被正确的编译。困难的部分仅限于特殊的小节，例如 :help 命令。

|<F1>| |:help| |:h| |help|
||:h[elp] {subject}|| +
||<F1>||
____________________________________________________________________________
Open help window.
The default section is shown unless {subject} is specified.
If you need help for a specific topic, try [c]:help overview[c].
____________________________________________________________________________

以上内容会被显示为：

<F1> :help :h help

:h[elp] {subject}

<F1>

Open help window. The default section is shown unless {subject} is specified. If you need help for a specific topic, try :help overview.

关于以上代码的一些说明：

  • |<F1>| 定义了一个 标签 。标签是洋红色的，会被显示在屏幕右侧，使用户可以根据它用 :help <F1> 跳转到这个主题。注意，写标签的时候必须从右到左，这样在例子中 |help| 才是最左边的标签而 |<F1>| 则是最右边的。

  • ||:h[elp] {subject}|| + 和 ||<f1>|| 是用于定义命令或映射的名称的，会被显示在屏幕的左侧。注意最后的加号，它表示换行。什么时候该换行而什么时候不该现在并没有特别的规定，你可以多试试，怎么好看怎么来。

  • 这条命令的实际帮助说明文字和之前的部分必须用至少一行四个下划线（_）分隔开来。这会产生引用效果和缩进，可以更清楚的表示出这些文字和它们说明的对象之间的关系。

  • 把条目包含在 [c] 中会使它们看起来像一条命令，同时还可以用 [o] 、 [m] 和 [a] 来包装选项、映射和参数。

  • 方便起见，以 {…} 、 [count] 和 [!] 开头的字符串都会被标记为一个参数。

可能用到的 Vimperator 文档的 asciidoc 专有命令还有：

  • section:撰写文档[writing-docs,documentation] 会创建一个新的章节 撰写文档 和两个标签 。

  • help:开发者信息[developer.html#documentation] 会创建一个名为 开发者信息 的链接，指向 developer.html 文件中的 documentation 标签。

如果你不知道应该将文档放在哪个文件或章节，请在邮件列表中或 IRC 的 #vimperator 频道询问。一般情况下，逻辑上相关的主题应该被放在一起，例如 打开网页 。

自动生成文档

 generating-docs

在你完成了一条新的命令、映射或选项之后，你也这样可以自动生成大部分 asciidoc 帮助文档：

:echo util.generateHelp(commands.get("addons"), "Extra text")

现在你可以复制并使用这些 asciidoc 文档，但请确认它们能够通过 "make doc" 编译并且看的过去，因为自动生成并不是那么可靠。
