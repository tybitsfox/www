styling

Vimperator 允许用户改变浏览器和任意网页的样式。所有的样式都通过 CSS 指定。尽管你可以通过 :style 命令指定任意用户界面元素的样式，大部分 Vimperator 元素的样式可以通过 :highlight 命令更加方便的进行修改。

E185 :colo :colorscheme

:colo[rscheme] {name}

加载一种配色方案。命令会从 'runtimepath' 选项指定的目录的 colors 子目录中加载第一个和方案名 {name} 相匹配的的 {name}.vimp 文件。

自动命令 ColorScheme 会在配色方案导入后被触发。

:hi :highlight

:hi[ghlight][!] [-append] {group}[{selector}] [{css}]

用 {css} 高亮 {group} 。一般情况下，Vimperator 会在应用 {css} 代码前对其进行语法检查。如果你确定代码的语法正确，可以使用 [!] 来跳过检查以加快 Vimperator 的加载速度。选择子 {selector} 可以是任意的合法 CSS 选择子，例如 :hover 。如果提供选择子，应用的对象将会局限于匹配的元素。

除非使用 -append 参数将 {css} 代码追加到现有的样式上，每次执行该命令都会使上次执行的效果消失。如果没有指定 {css} 代码，所有和 {group} 匹配的样式都会被列出。

:highlight-clear

:hi[light] clear [{group}[{selector}]]

:sty :style

:sty[le][!] [-name={name}] [-append] {filter} [{css}]

将 CSS 样式应用于浏览器或网页。过滤器 {filter} 是一列用于匹配的逗号分隔的 URL 地址。以 * 结尾的 URL 会被作为前缀进行匹配。不包含 : 或 / 的 URL 会被作为域名进行匹配。如果指定了名称 {name} （简写： -n ），已存在的同名样式会被覆盖。可以使用 -name 参数删除名为 {name} 的样式。如果同时提供了 -append （简写： -a ）和 -name 参数，{css} 和 {filter} 都会被追加到现有样式之上。

:dels :delstyle

:dels[tyle] [-name={name}] [-index={index}] [{filter}] [{css}]

  • -name ： 为 :style 命令指定的样式名（简写： -n ）

  • -index ： 对于匿名样式， :style 命令中列出的索引号（简写： -i ）
