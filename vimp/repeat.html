repeat.html

repeat

Vimperator 能够重复执行多个命令并录制宏。

宏

 complex-repeat macros

q

q{0-9a-zA-Z}

将一个键序列录制为一个宏。可用的宏名称为 {0-9a-zA-Z} （大写字母表示追加）。再次按下 q 会停止录制宏。

:macros

:mac[ros] [pat]

列出和正则表达式 [pat] 匹配的已录制的宏。如果没有指定正则表达式，列出所有宏。

:delmac :delmacros

:delmac[ros] {args}

:delmac[ros]!

删除和正则表达式 {args} 匹配的宏。使用 [!] 会删除所有的宏。

@ :play

:pl[ay] {arg}

[count]@{arg}

执行名为 {arg} 的宏 [count] 遍。映射 @ 仅接受 {0-9a-z} 作为参数 {arg} 。

@@

[count]@@

将最后一次执行的宏重放 [count] 遍。

.

[count].

将最后一次使用的键盘映射重复 [count] 遍。注意，和 Vim 不同的是，这不仅仅局限于编辑类命令，主要原因是 Vimperator 没有编辑类命令。

使用脚本

 using-scripts

:so :source

:so[urce][!] {file}

从文件 {file} 中导入 Ex 命令、Javascript 或是 CSS 。你可以导入包含例如 map < gt 的 Ex 命令的文件，把 Javascript 代码放在：

js <<EOF
hello = function () {
  alert("Hello world");
}
EOF

中，或是选择导入结尾为 .js 的文件。这种文件会自动被识别为纯 Javascript 代码文件。

注意：无论哪种情况你都必须如上所示为全局的 window 对象添加函数。函数：

function hello2() {
  alert("Hello world");
}

仅会在该脚本文件内有效。

用户家目录中的 .vimperatorrc 文件和 ~/.vimperator/plugin/ 目录下的所有文件都会在启动时被自动导入。~ 是访问 $HOME 目录的快捷键。使用 [!] 会压制出错信息。

:lpl :loadplugins

:loadplugins

立即加载所有未被加载的插件。因为插件会在 vimperatorrc 文件被导入后被自动导入，如果 vimperatorrc 中包含由插件实现的命令，此命令应该被包含在 vimperatorrc 文件中并在它们之前执行。另外，该命令也可以用来加载新安装的插件而无需重新启动 Vimperator 。

:ru :runtime

:runt[ime][!] {file} …

从 'runtimepath' 选项指定的目录中导入指定的文件。例如：
:runtime plugin/foobar.vimp

仅导入找到的第一个文件。使用 [!] 会导入所有找到的文件。

:scrip :scriptnames

:scrip[tnames]

按照第一次导入的先后顺序列出所有已导入的脚本文件名。

:fini :finish

:fini[sh]

停止导入一个脚本文件。该命令只能通过 Vimperator 脚本文件调用。

计时

 profiling profile

:time

:[count]time[!] {code|:command}

统计一条命令或一段代码运行所用的时间。将代码 {code} 运行 [count] 遍（缺省值：一遍）并返回所用的时间。代码 {code} 会由 Javascript 的 eval() 函数执行，所以可能有点慢，因此请对统计的结果有心理准备。

如果代码 {code} 是以 : 开头的，它将会被作为 Vimperator 的一条命令被执行。

如果你仅想多次执行某条命令而不需要显示统计数据，请使用 [!] 。
