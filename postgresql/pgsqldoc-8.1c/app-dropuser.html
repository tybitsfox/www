dropuser

Name

dropuser -- 删除一个 PostgreSQL 用户帐户

Synopsis

dropuser [option...] [username]

描述

dropuser 删除一个现有 PostgreSQL 用户 和 该用户所有的数据库。 只有超级用户（在 pg_shadow 表里面设置了 usesuper 的用户）可以创建新的 PostgreSQL 用户。

dropuser 是 SQL 命令 DROP USER的封装。 因此，用哪种方法删除用户没什么区别。

选项

dropuser 接受下列命令行参数：

username

声明要删除的 PostgreSQL 用户名。 如果你没有在命令行上声明，将会被提示输入一个。

-e
--echo

回显 dropuser 生成的命令并发送给服务器。

-i
--interactive

在真正删除用户前前提示。

-q
--quiet

不显示响应。

createuser 还接受下列命令行参数做连接参数：

-h host
--host host

声明运行服务器的机器的主机名。 如果主机名以斜杠开头，则它被用做到 Unix 域套接字的路径。

-p port
--port port

声明服务器正在侦听的互联网 TCP 端口号或本地Unix域套接字的文件扩展（描述符）。

-U username
--username username

进行连接的用户名（不是要删除的用户名）。

-W
--password

强制口令提示（要连接到服务器，不是要删除的用户的口令）。

环境

PGHOST
PGPORT
PGUSER

缺省连接参数

诊断

如果出现错误，将会显示后端错误信息。参阅 DROP USER 和psql获取可能信息描述。 数据库服务器必须在目标主机上运行。同样，任何 libpq 前端库可获得的缺省设置和环境变量都将生效。

例子

删除缺省数据库服务器上的用户 joe： 

$ dropuser joe
DROP USER

用在主机eden上的服务器删除用户joe， 端口是 5000，执行命令前提示并且显示执行的命令： 

$ dropuser -p 5000 -h eden -i -e joe
User "joe" and any owned databases will be permanently deleted.
Are you sure? (y/n) y
DROP USER "joe"
DROP USER

又见

createuser, DROP USER
